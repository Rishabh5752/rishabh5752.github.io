Associate

Bain and Company, Gurugram

I am currently building ExcelGPT, an agentic GenAI application that lets users interrogate Excel files in plain English. The core of the system is a multi-LLM pipeline: when a file is uploaded, an LLM analyses each column to classify its purpose, builds a cross-sheet formula dependency graph, and produces structural skeletons of the data. When a query arrives, the system classifies intent and routes it through a three-stage pipeline that prunes relevant tables from the full inventory, scores chunks for relevance in parallel, and generates DuckDB SQL with automatic self-repair on failure.

The central design insight is treating every Excel cell as a string and resolving types at query time using TRY_CAST, which makes the system robust to the kind of mixed, malformed data that appears in real-world spreadsheets. For queries that return more data than fits in context, a synthesis layer answers the result in segments and consolidates them into a single plain-English response with source attribution.
